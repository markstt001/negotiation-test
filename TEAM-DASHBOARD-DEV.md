## 📊 专业分析逻辑

### 1. 团队类型判定

| 类型 | 判定条件 | 描述 |
|------|----------|------|
| ⚖️ 均衡型 | 最大占比 - 最小占比 < 15% | 各维度分布均匀，能力全面 |
| 🧠 分析型 | A 维度 > 60% | 擅长数据分析和逻辑推理 |
| 💡 直觉型 | I 维度 > 60% | 擅长创新洞察和机会识别 |
| 🤝 关系型 | R 维度 > 60% | 擅长关系维护和协调沟通 |
| 🎯 任务型 | T 维度 > 60% | 擅长目标达成和高效执行 |
| ⚔️ 竞争型 | C 维度 > 60% | 擅长争取利益和谈判博弈 |
| 🤲 合作型 | B 维度 > 60% | 擅长建立长期合作关系 |
| 🛡️ 防御型 | D 维度 > 60% | 擅长风险控制和谨慎决策 |
| 🚀 开拓型 | P 维度 > 60% | 擅长创新尝试和快速行动 |
| 🔀 互补型 | 其他情况 | 成员风格多样，优势互补 |

### 2. 优势分析逻辑

```javascript
// 四维度两两对比，生成优势描述
if (A >= I && A > 0) → "数据分析能力强"
else if (I > 0) → "创新洞察能力强"

if (R >= T && R > 0) → "关系维护能力好"
else if (T > 0) → "任务执行能力强"

if (C >= B && C > 0) → "竞争意识强"
else if (B > 0) → "合作共赢意识好"

if (D >= P && D > 0) → "风险控制能力强"
else if (P > 0) → "开拓创新能力好"
```

### 3. 风险识别逻辑

```javascript
// 检测维度缺失或不足
if (A === 0 && I === 0) → "缺少数据分析者"
else if (A < 2 && I < 2) → "数据分析能力弱"

// ... 其他维度同理

// 风险描述包含具体后果
"容易凭感觉或经验决策，可能忽略关键数据"
```

### 4. 成员对比分析

**互补性计算:**
```javascript
var complementCount = 0;
if (code1[0] !== code2[0]) complementCount++; // A/I
if (code1[1] !== code2[1]) complementCount++; // R/T
if (code1[2] !== code2[2]) complementCount++; // C/B
if (code1[3] !== code2[3]) complementCount++; // D/P

// 4 个维度都不同 → 高度互补
// 2-3 个维度不同 → 中度互补
// 1 个维度不同 → 轻度互补
// 0 个维度不同 → 风格相似
```

**协作建议生成:**
- 根据双方风格特点，推荐分工方案
- 识别潜在冲突点，提供沟通建议
- 针对相同风格，提醒避免盲点重叠

## 🔧 核心函数

| 函数名 | 功能 | 参数 |
|--------|------|------|
| `goToTeamDashboard()` | 进入团队仪表盘 | 无 |
| `closeTeamDashboard()` | 关闭仪表盘 | 无 |
| `renderTeamDashboard(team)` | 渲染仪表盘 | 团队对象 |
| `switchTeamTab(tabName)` | 切换 Tab | 'overview'/'analysis'/'compare'/'report' |
| `runMemberComparison()` | 执行成员对比 | 无（从 DOM 读取） |
| `generateMemberComparison(m1, m2)` | 生成对比分析 | 2 个成员对象 |
| `addTeamMember()` | 添加成员 | 无（弹窗输入） |
| `inviteTeamMembers()` | 邀请成员 | 无（调用分享 API） |
| `unlockTeamReport()` | 解锁报告 | 无（模拟支付） |
| `calculateStyleDistribution(members)` | 计算风格分布 | 成员数组 |
| `calculateDimensionDistribution(members)` | 计算维度分布 | 成员数组 |
| `analyzeTeamType(dimensions, total)` | 分析团队类型 | 维度对象 + 总数 |
| `generateTeamAdvantages(dimensions)` | 生成优势列表 | 维度对象 |
| `generateTeamRisks(dimensions)` | 生成风险列表 | 维度对象 |

## 🧪 测试流程

### 1. 基础功能测试
```
1. 打开 index.html
2. 完成测试，获得风格代码
3. 点击"创建团队"，输入团队名称
4. 点击"团队仪表盘"按钮
5. 验证概览页显示正确
```

### 2. 分析功能测试
```
1. 在仪表盘中切换到"分析"Tab
2. 验证四维度分析显示正确
3. 验证团队优势生成合理
4. 验证风险识别准确
```

### 3. 对比功能测试
```
1. 添加至少 2 位成员（手动或示例）
2. 切换到"对比"Tab
3. 选择两位成员
4. 验证对比结果生成
5. 测试不同组合的互补性分析
```

### 4. 报告功能测试
```
1. 添加至少 3 位成员
2. 切换到"报告"Tab
3. 验证免费预览内容
4. 点击"立即解锁"，确认支付提示
```
